MedPlatform — Deployment Guide

Full-stack deploy (React + Node.js + MongoDB) for a 2–3 day preview window

Railway · MongoDB Atlas · Always-on · Free

Never commit .env to GitHub. Add .env to your .gitignore before pushing. All secrets (JWT keys, API keys) must be set as environment variables on the hosting platform.

Add .gitignore & push to GitHub

Make sure .env files are excluded, then push the repo.

# From the project root
echo ".env" >> .gitignore
echo "node_modules/" >> .gitignore

git init
git add .
git commit -m "initial commit"

# Create a new repo on github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/medplatform.git
git push -u origin main
1

Create a free MongoDB Atlas cluster

  1. Go to mongodb.com/atlas → create a free account
  2. Create a new project → build a Free (M0) cluster
  3. Under Database Access, add a user with a strong password
  4. Under Network Access, add IP 0.0.0.0/0 (allow all — fine for a short demo)
  5. Click Connect → Drivers → copy the connection string:
mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/BuildathonDB?retryWrites=true&w=majority

Save this string — you'll use it as MONGODB_URI in the next steps.

2

Deploy on Railway

Go to railway.app → sign in with GitHub → New Project → Deploy from GitHub repo → select medplatform.

Railway will create one service by default. You need two services — add the second via + New Service → GitHub Repo (same repo, different root dir).

Service 1 — Backend
  • Root dirserver
  • Buildnpm install && npm run build
  • Startnpm start
Service 2 — Frontend
  • Root dirclient
  • Buildnpm install && npm run build
  • Startnpx serve dist -p $PORT

3

Seed the database

After the backend service is running, open the Railway dashboard → backend service → Settings → Shell tab and run:

npm run seed

This populates doctors, patients, and Q&A data.

4

Fix the Vite proxy for production

The current vite.config.ts proxies /api to localhost:3001 — this only works locally. In production the frontend must call the real backend URL. Two options:

Option A — use VITE_API_URL in fetch calls (preferred)

Wherever your frontend calls the API, prefix with the env variable:

// Before
fetch('/api/doctors')

// After
fetch(`${import.meta.env.VITE_API_URL}/api/doctors`)

Option B — keep the proxy, serve from same domain

Build the React app into the server's public/ folder and serve both from the same Railway service. Simpler but requires adding static-file serving to Express.

Teardown after 2–3 days

To stay within the free tier and avoid any charges:

  1. Railway dashboard → your project → Settings → Delete Project
  2. MongoDB Atlas → Clusters → Terminate (or just leave the free M0 — it never expires)
