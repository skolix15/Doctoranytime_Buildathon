Create your .env file

Copy the example file and fill in your Anthropic API key and JWT secrets:

cp .env.example .env
VariableValue
JWT_SECRETAny random 32+ character string
JWT_REFRESH_SECRETAny different random 32+ character string
ANTHROPIC_API_KEYYour sk-ant-... key
VITE_API_URLhttp://localhost:3001 for local; public backend URL for remote
CLIENT_URLhttp://localhost:5173 for local; public frontend URL for remote

MongoDB credentials (admin / vamos) are already wired between the mongo and server services in docker-compose.yml — no changes needed for local runs.

1

Start all services with Docker Compose

The repo includes a docker-compose.yml that spins up three containers: mongo, server (Node.js), and client (React + nginx). Requires Docker Desktop or Docker Engine with the Compose plugin.

# Build images and start in detached mode
docker compose up --build -d
server — Node.js :3001
  • Imagenode:20-alpine (multi-stage)
  • Port3001 → 3001
  • DBmongo container (internal)
client — nginx :5173
  • Imagenginx:alpine (static build)
  • Port5173 → 80
  • Routingtry_files for React Router

After startup, check all three containers are healthy:

docker compose ps

Seed the database

docker compose exec server node dist/scripts/seed.js

Demo script — key features to show

  • 🏠
    Dashboard — AI health insights card (pre-generated), upcoming appointment cards, blood pressure chart. If an appointment is within 24h, an amber briefing banner appears with one-tap AI prep.
  • 🔍
    Search — type a symptom or specialty; doctor cards with match scores appear alongside Q&A results. Click a doctor card → slide-in drawer with full profile, availability, and booking CTA. Auto-searches when redirected from the AI assistant.
  • 🤖
    Medical Assistant — streaming chat grounded in the patient's real data. Past sessions in the left sidebar. Source pills on messages open the full Q&A thread. Booking card appears when AI recommends a specialist — shows previous doctors or redirects to search pre-filled with the specialty.
  • 💬
    Assistant Popup — click 🧠 on any medication, test result, or appointment card to open the AI popup in the bottom-right corner without leaving the page. Contextual prefill sent automatically.
  • 💊
    Health Records — three tabs: Medications (AI "why am I taking this?" accordion), Test Results (structured values, status badges, file attachments), Documents (vault with AI summaries).
  • 📅
    Health History — four tabs: upcoming, past, all appointments, and a bar chart of appointments by month. Past appointments show doctor notes, diagnosis, and prescriptions.
  • 👨‍👩‍👧
    Family Hub — add family members with tabbed Profile + Clinical form. Switch active member to scope the AI to that person's conditions.

Stop / teardown

# Stop containers (keeps data volume)
docker compose down

# Stop and delete everything including the MongoDB data volume
docker compose down -v

Alternative — deploy publicly on Railway

If you need a public URL (e.g. to share with judges), Railway can host all three services for free from the same GitHub repo. The Dockerfiles in server/ and client/ are already Railway-compatible.

  1. Push the repo to GitHub
  2. Go to railway.app → New Project → Deploy from GitHub
  3. Add three services: one pointing to server/, one to client/, and one using the MongoDB template from the Railway marketplace
  4. On the server service, set all the same env vars from Step 0, plus MONGODB_URI from the Railway MongoDB service (Railway injects this automatically if you link the services)
  5. On the client service, set VITE_API_URL to the Railway backend public URL
  6. Railway generates public domains under Settings → Networking → Generate Domain for each service
  7. Run npm run seed once via the Railway backend shell
