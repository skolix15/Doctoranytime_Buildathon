MedPlatform — Deployment Guide

Full-stack deploy (React + Node.js + MongoDB) for a 2–3 day preview window

Railway · MongoDB Atlas · Always-on · Free tier
⚠️
Never commit .env to GitHub. Add .env to your .gitignore before pushing. All secrets (JWT keys, Anthropic API key) must be set as environment variables on the hosting platform — never hardcoded.
0

Add .gitignore & push to GitHub

Make sure .env files and build artifacts are excluded, then push the repo.

# From the project root
git init
git add .
git commit -m "initial commit"

# Create a new repo on github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/medplatform.git
git push -u origin main

Verify your .gitignore at the root includes node_modules/, dist/, and **/.env.

1

Create a free MongoDB Atlas cluster

  1. Go to mongodb.com/atlas → create a free account
  2. Create a new project → build a Free (M0) cluster (512 MB — more than enough for seed data)
  3. Under Database Access, add a user (e.g. admin) with a strong password
  4. Under Network Access, add IP 0.0.0.0/0 (allow all — fine for a short demo)
  5. Click Connect → Drivers → copy the connection string
mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/BuildathonDB?retryWrites=true&w=majority

Save this string — you'll use it as MONGODB_URI in the next step.

2

Deploy on Railway

Go to railway.app → sign in with GitHub → New Project → Deploy from GitHub repo → select medplatform.

Railway creates one service by default. You need two services — add the second via + New Service → GitHub Repo (same repo, different root dir).

Service 1 — Backend
  • Root dirserver
  • Buildnpm install && npm run build
  • Startnpm start
Service 2 — Frontend
  • Root dirclient
  • Buildnpm install && npm run build
  • Startnpx serve dist -p $PORT

3

Fix the Vite proxy for production

The vite.config.ts dev proxy (/api → localhost:3001) is local-only. In production all API calls must go to the real backend URL. Two options:

Option A — VITE_API_URL prefix (preferred)

The API client (client/src/api/) should prefix every request with the env variable:

// axios baseURL or fetch prefix
const BASE = import.meta.env.VITE_API_URL ?? ''
axios.defaults.baseURL = `${BASE}/api/v1`

Option B — serve both from one Railway service

Build the React app into server/public/ and serve static files from Express. Simpler billing (one service) but couples frontend and backend deploys.

Demo script — key features to show

  • 🏠
    Dashboard — AI health insights card (pre-generated), upcoming appointment cards, blood pressure chart. If an appointment is within 24h, an amber briefing banner appears with one-tap AI prep.
  • 🔍
    Search — type a symptom or specialty; doctor cards with match scores appear alongside Q&A results. Click a doctor card → slide-in drawer with full profile, availability, and booking CTA. Auto-searches when redirected from the AI assistant.
  • 🤖
    Medical Assistant — streaming chat grounded in the patient's real data. Past sessions in the left sidebar. Source pills on messages open the full Q&A thread. Booking card appears when AI recommends a specialist — shows previous doctors or redirects to search pre-filled with the specialty.
  • 💬
    Assistant Popup — click 🧠 on any medication, test result, or appointment card to open the AI popup in the bottom-right corner without leaving the page. Contextual prefill sent automatically.
  • 💊
    Health Records — three tabs: Medications (AI "why am I taking this?" accordion), Test Results (structured values, status badges, file attachments), Documents (vault with AI summaries).
  • 📅
    Health History — four tabs: upcoming, past, all appointments, and a bar chart of appointments by month. Past appointments show doctor notes, diagnosis, and prescriptions.
  • 👨‍👩‍👧
    Family Hub — add family members with tabbed Profile + Clinical form. Switch active member to scope the AI to that person's conditions.
!

Teardown after demo

To stay within free tier limits and avoid charges:

  1. Railway dashboard → your project → Settings → Delete Project
  2. MongoDB Atlas → Clusters → Terminate (or leave the free M0 — it never expires and costs nothing)
